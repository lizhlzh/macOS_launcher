三、修复拖拽排序卡顿，但保留动画

当前问题：
LauncherStore.previewMoveTile 每次 hover 到新目标都会触发：
onChange?(.content(animated: true))

这会导致 LauncherRootView.reload(animated: true) -> LauncherPagerView.performReload(animated: true...)。拖拽过程中频繁整页 reload、重排、动画、刷新页面副本，所以卡顿。

目标：
保留拖拽排序动画，但不要使用完整 content reload。实现专用轻量拖拽排序预览动画：只移动已有 TileView 的 frame，不重建 Tile，不刷新图标，不刷新 folder icon，不刷新 edge replicas，不重新生成快照。

三点原则：
1. 普通内容刷新仍可使用 .content(animated: true)。
2. 拖拽排序预览必须使用单独的 .dragPreview。
3. .dragPreview 只做已有 TileView 的位置过渡动画。

七、LauncherPagerView 增加轻量拖拽动画

文件：
Sources/MacOSLauncher/Features/Launcher/UI/LauncherViews.swift

在 LauncherPagerView 中新增状态：

private var dragPreviewScheduled = false
private var needsDragPreviewAfterCurrentPass = false

新增方法：

func scheduleDragPreviewAnimation() {
    if dragPreviewScheduled {
        needsDragPreviewAfterCurrentPass = true
        return
    }

    dragPreviewScheduled = true

    DispatchQueue.main.async { [weak self] in
        guard let self else { return }

        self.dragPreviewScheduled = false
        self.applyDragPreview()

        if self.needsDragPreviewAfterCurrentPass {
            self.needsDragPreviewAfterCurrentPass = false
            self.scheduleDragPreviewAnimation()
        }
    }
}

新增 applyDragPreview：

func applyDragPreview() {
    guard bounds.width > 0, bounds.height > 0 else { return }

    let tiles = store.visibleTiles
    var movements: [(view: LauncherTileView, frame: NSRect)] = []

    for (index, tile) in tiles.enumerated() {
        guard let view = tileViews[tile.id] else {
            continue
        }

        let pageIndex = index / metrics.itemsPerPage
        guard let pageView = pageViews[pageIndex] else {
            continue
        }

        let targetFrame = frameForTile(at: index)

        if view.superview !== pageView {
            let currentFrameInWindow = view.convert(view.bounds, to: nil)
            pageView.addSubview(view)
            view.frame = pageView.convert(currentFrameInWindow, from: nil)
        }

        if view.tileID == store.draggedTileID {
            view.frame = targetFrame
            continue
        }

        if view.frame.integral != targetFrame.integral {
            movements.append((view, targetFrame))
        }
    }

    guard !movements.isEmpty else {
        return
    }

    NSAnimationContext.runAnimationGroup { context in
        context.duration = 0.11
        context.timingFunction = CAMediaTimingFunction(
            controlPoints: 0.18,
            0.82,
            0.22,
            1.0
        )

        for movement in movements {
            movement.view.animator().frame = movement.frame
        }
    }
}

说明：
- 被拖拽的 TileView 不参与动画，直接移动到目标 frame；
- 其他被挤开的 TileView 做 0.11 秒轻量 frame 动画；
- 不调用 performReload；
- 不刷新 edge replicas；
- 不刷新图标；
- 不重新生成页面快照；
- 不重新创建 TileView。

八、拖拽期间关闭 page rasterization，结束后恢复

当前 pageView 使用 shouldRasterize。静态展示可能有用，但拖拽时子视图持续移动，页面层不断重新栅格化可能更卡。

在 LauncherPagerView 的 tileViewDidBeginDragging 中：

func tileViewDidBeginDragging(_ view: LauncherTileView) {
    setPageRasterizationEnabled(false)
    store.beginDraggingTile(view.tileID)
}

在 tileViewDidEndDragging 中：

func tileViewDidEndDragging(_ view: LauncherTileView) {
    dropTargetID = nil
    store.endDraggingTile(commit: false)
    setPageRasterizationEnabled(true)
    refreshEdgeReplicas()
}

在 performDropWith 的 defer 中也恢复：

defer {
    dropTargetID = nil
    store.endDraggingTile(commit: true)
    setPageRasterizationEnabled(true)
    refreshEdgeReplicas()
}

如果 performDragOperation 里也有 defer，同样恢复：

defer {
    dropTargetID = nil
    store.endDraggingTile(commit: true)

要求：
- 拖拽过程中不要 refreshEdgeReplicas；
- 拖拽过程中不要 snapshot；
- 拖拽结束后再刷新 edge replicas，保证分页视觉不坏。

九、修复 draggingSession endedAt 和 performDrop 双重 endDragging 的风险

当前 Tile 作为 dragging source，endedAt 里会调用 delegate?.tileViewDidEndDragging(self)。如果 drop 成功路径里 performDropWith 已经调用 store.endDraggingTile(commit: true)，endedAt 又调用 commit: false，可能存在状态回滚或重复结束风险。

要求：
1. 增加一个拖拽完成状态，避免 drop 成功后 endedAt 再回滚。
2. 在 LauncherTileView 或 LauncherPagerView 中记录当前 drag 是否已经 committed。
3. performDropWith 成功后标记 committed。
4. draggingSession endedAt 如果 operation == .move 且已经由 performDropWith 处理，不要再 commit:false。

简单方案：

在 LauncherPagerView 中增加：

private var currentDragCommitted = false

tileViewDidBeginDragging：

currentDragCommitted = false
setPageRasterizationEnabled(false)
store.beginDraggingTile(view.tileID)

performDropWith defer 中：

currentDragCommitted = true
dropTargetID = nil
store.endDraggingTile(commit: true)
setPageRasterizationEnabled(true)
refreshEdgeReplicas()

tileViewDidEndDragging：

dropTargetID = nil
if !currentDragCommitted {
    store.endDraggingTile(commit: false)
}
currentDragCommitted = false
setPageRasterizationEnabled(true)
refreshEdgeReplicas()

注意：
- 如果实际 AppKit 调用顺序是 endedAt 在 performDropWith 后，不能回滚成功 drop。
- 如果拖拽取消或放到无效位置，仍然 commit:false 回滚预览。
